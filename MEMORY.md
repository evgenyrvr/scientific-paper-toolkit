# Scientific Paper AI Lab — Project Memory

This file stores generic learnings that persist across sessions. It loads into the model's context automatically. Keep under 200 lines.

## Workflow learnings

- [LEARN:workflow] Start every session with: read status_dashboard + last iteration report. This takes one exchange and prevents duplicating work.
- [LEARN:workflow] One iteration = one small task. If the task requires reading more than 5 files, split it.
- [LEARN:workflow] When context compacts, the model starts cold. The session_log and status_dashboard are the recovery points.
- [LEARN:anti-cheating] Never accept "verified" without a concrete verification table row. Verbal confirmation is not verification.
- [LEARN:claims] Add to claims register BEFORE writing it into the manuscript, not after.
- [LEARN:references] Never accept a DOI or arXiv ID from model memory alone. Confirm from publisher page.

## Physics-specific learnings

- [LEARN:physics] Dimensional analysis is mandatory before a formula goes into the manuscript.
- [LEARN:physics] Limiting cases (b→0, b→∞, a_s→0, M→0) must be checked for every new formula.
- [LEARN:physics] Point-mass formulas diverge at b=0; extended-profile formulas are regular. Check which limit applies before using either.
